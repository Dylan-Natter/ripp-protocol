## Getting Help

### Questions and Discussions

For general questions about RIPP:

1. Check the [FAQ](https://dylan-natter.github.io/ripp-protocol/faq)
2. Browse [existing issues](https://github.com/Dylan-Natter/ripp-protocol/issues)
3. Open a new issue with the "Question" label

### Bug Reports

If you've found a bug:

1. Search [existing issues](https://github.com/Dylan-Natter/ripp-protocol/issues) to avoid duplicates
2. Use the [Bug Report template](https://github.com/Dylan-Natter/ripp-protocol/issues/new?template=bug_report.yml)
3. Provide clear steps to reproduce the issue
4. Include relevant error messages and environment details

### Feature Requests

To propose a new feature:

1. Search [existing issues](https://github.com/Dylan-Natter/ripp-protocol/issues) for similar requests
2. Use the [Feature Request template](https://github.com/Dylan-Natter/ripp-protocol/issues/new?template=feature_request.yml)
3. Explain the use case and expected behavior

### Specification Changes

To propose changes to the RIPP specification:

1. Use the [Spec Change template](https://github.com/Dylan-Natter/ripp-protocol/issues/new?template=spec_change.yml)
2. Provide rationale and real-world examples
3. Allow time for community discussion

## Community Guidelines

Please follow our [Code of Conduct](CODE_OF_CONDUCT.md) when interacting with the community.

## Response Time

- **Bug reports**: Typically reviewed within 48 hours
- **Feature requests**: Discussed within 1 week
- **Spec changes**: Require community consensus (2+ weeks)

Note: This is an open source project maintained by volunteers. Response times may vary.
